# SENSAI (센세이) 웹사이트 구조 및 기능 정의서 (Information Architecture)

이전 UX/UI 레퍼런스 분석이 **'시각적 인터랙션과 사용자 경험(디자인)'**에 집중되었다면, 본 문서는 **'실제 웹사이트의 메뉴 구성, 페이지 이동 흐름(User Flow), 관리자(Admin) 기능'** 등 실질적인 구조를 정의합니다.

사업계획서 상 센세이의 타겟은 **100% B2B (기업, 지자체 축제 등)** 이므로, 불필요한 일반 유저용 '마이페이지' 보다는 **'강력한 포트폴리오 쇼케이스'**와 **'원활한 도입 문의(세일즈) 및 관리'**에 초점을 맞추어 설계했습니다.

---

## 1. 프론트엔드 사이트맵 (사용자 노출 페이지)

B2B 파트너(클라이언트)가 센세이의 미디어아트를 확인하고, 도입을 문의하기까지의 여정을 최적화한 구조입니다.

### 1.1. Home (메인 페이지)
* **Hero Visual:** 전체 화면 인터랙티브 비디오 (팀랩/프레임리스 벤치마킹)
* **Featured IPs:** 현재 가장 주력하는 미디어아트 IP 2~3종 하이라이트 미리보기
* **Value Proposition:** 센세이가 제공하는 B2B 현장 체험 콘텐츠의 핵심 가치 요약
* **Quick Contact:** 하단에 즉시 문의를 남길 수 있는 플로팅 버튼 또는 미니 폼

### 1.2. Works / Portfolio (프로젝트 & IP 갤러리)
* **IP List (목록):** 지금까지 개발된 미디어아트 작품 IP들을 썸네일/비디오 그리드 형태로 나열. (카테고리 필터링: 전시용, 축제용, 팝업스토어 등)
* **Project Detail (상세 페이지):** 
  - 특정 IP 클릭 시 이동. 
  - 작품 설명, 도입 시 공간 요구사항(평수, 필요 장비), 실제 현장 설치 및 관람객 상호작용 영상 제공.
  - 하단에 `[이 작품 도입 문의하기]` CTA 버튼 배치.

### 1.3. About (기업 소개)
* **Vision & Story:** 일해라컴퍼니와 센세이 프로젝트의 비전 (기술과 예술의 융합)
* **Team:** 대표자 및 핵심 팀원(미디어아트 연출, AI 개발 등) 역량 소개
* **Tech Stack:** 사용되는 AI 비전 기술, 센서 등 기술적 강점 어필

### 1.4. Contact / B2B Inquiry (도입 문의)
* **문의 폼 (Inquiry Form):** 기업명, 담당자, 연락처, 희망 설치 공간 크기, 원하시는 IP 종류, 예산 범위를 입력할 수 있는 폼.
* **B2B 제안서 다운로드:** 기본 B2B 표준 제안 패키지를 다운로드할 수 있는 링크 제공 (이메일 수집 후 다운로드 가능하도록 설계하여 리드 확보 가능).

## 2. 백오피스 / 관리자 페이지 (Admin Page)

웹사이트 운영자(센세이 팀)가 개발자의 도움 없이도 지속적으로 콘텐츠를 업데이트하고, 들어온 고객 문의를 관리할 수 있는 내부 시스템입니다.

### 2.1. Dashboard (대시보드)
* **요약 통계:** 일간/주간 사이트 방문자 수, 신규 B2B 도입 문의 건수 요약.

### 2.2. Portfolio / IP Management (포트폴리오 관리)
* **작품 등록/수정/삭제 (CRUD):** 
  - 새로운 미디어아트 IP가 개발될 때마다 제목, 설명, 썸네일 이미지, 배경 비디오 등을 직접 업로드하고 배포.
  - 노출 순서 변경 및 카테고리 설정.

### 2.3. Inquiry Management (문의 및 영업 리드 관리)
* **문의 목록 (Inbox):** Contact 페이지를 통해 들어온 B2B 클라이언트의 문의 내역 리스트업.
* **상세 확인 및 상태 변경:** 문의 내용 상세 확인 기능 및 처리 상태(신규 대기, 연락 완료, 계약 진행중, 거절 등) 변경을 통한 영업 파이프라인 관리.

### 2.4. Site Settings (사이트 설정)
* **메인 비주얼 관리:** 메인 홈페이지의 전체화면 백그라운드 영상 교체 기능.
* **회사 정보 관리:** 하단 Footer에 들어갈 연락처, 이메일, 주소 등 변경 기능.

---

## 3. 결론: 일반 유저용 '마이페이지' 생략 사유

센세이의 사업계획서에 따르면 사이트의 목표는 **'B2B 제안 패키지 구축'** 및 **'대중견 기업협력/지자체 도입'**입니다. 
일반 관람객이 회원가입을 하고 굿즈나 티켓을 개별 구매하는 B2C 커머스 형태가 아니기 때문에, 초기에 로그인/마이페이지 기능을 구축하는 것은 리소스 낭비일 수 있습니다.

따라서 1차 홈페이지는 **1) 완벽한 포트폴리오 쇼룸(프론트엔드)**과 **2) 효율적인 문의 관리 및 작품 업로드(관리자 페이지)**의 2-Track 구조로 가는 것을 제안합니다.
